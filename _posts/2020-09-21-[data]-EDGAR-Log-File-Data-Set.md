# EDGAR?
EDGAR이란 전자 데이터 수집, 분석 및 검색 시스템이다. EDGAR는 미국 증권 거래위원회에 제출하기 위하여 회사 및 기타 업체의 요구되는 제출물을 법률에 의해 자동으로 수집, 확인, 색인 생성, 수락 및 전달을 수행한다.

---

# EDGAR Log File Data?
DERA(Division of Economic and Risk Analysis, DERA)[^1]는 SEC.gov[^2] 를 통해 EDGAR 파일링에 대한 인터넷 검색 트래픽 정보를 2003년 2월 14일부터 2017년 6월 30일까지의 기간동안 수집했다. 이 데이터는 공개적으로 접근이 가능한 EDGAR 사용에 대한 단순하지만 광범위한 통찰력을 제공한다.

[^1]: 경제 위험 분석 보고
[^2]: SEC.gov는 한국으로 치면 금감원정도 되겠다.


EDGAR Log File Dataset 은 SEC.gov 웹 사이트의 사용자 액세스 통계를 기록하고 저장하는 Apache 로그 파일에서 추출한 CSV 형식의 정보가 포함되어 있다. 손실되거나 손상된 파일의 존재를 포함한 특정 제한에 의해 수집된 정보는 모든 SEC.gov 웹사이트 트래픽을 캡처하지 못할 수 있다. 또한, 데이터 추출 및 데이터 취합 과정에서 데이터에 부정확하거나 다른 오류가 유입되었을 수 있다. 100만 개 이상의 항목을 포함할 수 있는 데이터 파일의 큰 크기를 감안할 때 최상의 결과를 위해서는 데이터 양을 전부 불러 올 수 없는 소프트웨어는 사용하지 않는 것이 좋다.

# 데이터 구성
- "logyyyymmdd.zip" 형태로 2003년 1월 1일부터 2017년 6월 30일까지의 압축파일로 구성되어 있다.
- 압축 파일을 열면 로그 데이터가 담긴 csv 파일과 각 컬럼의 속성을 알려주는 README.txt 파일로 구성되어 있다.

---

# 데이터 크기
- 2003년 1월 1일 파일에는 데이터가 없다.
- 2003년 1월 2일 파일에는 row가 세줄 밖에 없다.(1Kb)
- 2003년 2월 14일 본격적으로 데이터 수집을 시작한 시점에는 128,905개의 row가 있다.(14.3Mb)
- 2010년 12월 31일 대략적인 중간 시점에는 1,048,576개의 row가 있다.(265Mb)
- 2017년 6월 30일 log는 excel로는 열리지 않을 만큼 용량이 크다.(2.6Gb)
- 최근 데이터일수록 평균적으로 용량이 커질 것으로 예상됨

# 컬럼 속성과 의미
1. IP adress : 완전한 ip 구조가 아닌 마지막 주소의 고유성을 보존하는 3자리 문자열로 구성되어 있다. (ex : 207.17.47.egd) 만약 마지막 주소가 만일 150이면 같은 문자열로 되어있다는 뜻.

2. date : yyyy-mm-dd 형태로 파일명과 동일한 날짜.(Apache log file date)

3. time : 00:00:00부터 23:59:59까지 log가 찍힌 시간.(Apache log file time)

4. zone : 요청 처리를 완료한 서버와 관련된 시간대를 말한다.

5. cik : SEC에서 요청한 문서와 연관된 키 (central index key)

6. accession : 요청한 문서와 연관된 SEC 문서 등록 번호

7. doc : 확장자를 포함한 요청된 파일의 파일명. 만약 파일 이름이 없고 확장자 명만 있는 경우는 문서 등록 번호가 파일 이름이다.

8. code : 응답, 리디렉션, 오류 등 요청과 관련된 HTTP 상태코드

9. filesize : 문서 파일의 용량

10. idx : 요청자가 문서의 인덱스 페이지의 머무르는 경우 1의 값을 갖는다.(ex : index.htm), 그 외 경우에는 0

11. norefer : Apache 로그 파일 Referrer 필드가 비어 있으면 값을 1로, 그 외 경우에는 0

12. noagent : Apache 로그 파일 사용자 에이전트 필드가 비어있으면 값을 1로, 그 외 경우에는 0

13. find : 0에서 10까지의 숫자값. Referrer 필드에서 'string/[$string]/'이 발견 되었는지에 해당하며, 이는 문서 요청자가 문서 링크에 도착한 방법을 나타낼 수 있다.(ex : 내부 EDGAR 검색)
