# 데이터 모델링(개념적, 논리적, 물리적) 개념과 절차, ER 표기법
## 데이터 모델링 절차
- 특징 : 추상화, 단순화 명확화
- 요구사항의 모호성을 제거
프로세스 모델링 : 목표시스템에서 요구하는 기능

### 개념적 모델링
- 현실세계에 나타난 것을 구조화시키는 단계(추상적으로 개념화한다.)
- 핵심 엔터티와 그들 간의 관계를 발견하는 단계
- 엔터티, 엔터티의 식별자, 엔터티 간 관계, 관계의 대응 수 및 차수, 엔터티의 속성등이 도출됨
- 엔터티 = 객체
- 릴레이션 = 테이블
- 속성 열 = 컬럼
- 전체 데이터 = 인스턴스
- 하나의 데이터 행 = 튜플
- 차수 = 컬럼 수

### 논리적 모델링
- 좀 더 명확하게 만드는 단계
- 논리적으로 구체화 시키는 단계
- ERD를 그리는 단계


### 물리적 모델링
- 열의 데이터 형식, 제약조건, 인덱스가 정의됨
- 컴퓨터 하드웨어 시스템에 어떤식으로 표현할지 다루는 단계

# 함수종속성, 암스트롱의 원칙. DB 이상현상
## 이상현상
### 삽입 이상
- 원하지 않는 정보까지 함께 삽입해야만 하는 상황
### 삭제 이상
- 필요한 정보까지 함께 삭제되어야 하는 현상
### 수정 갱신
- 동일한 내용을 여러건의 데이터에서 반복 수정해야하는 현상

## 함수종속성
- X -> Y
- 결정자 -> 종속자
- Y는 X에 종속한다.

- 1차 정규화
- 1NF

# 트랜잭션의 개념과 직렬성
### 개념
- 업무처리 단위

- ACID
  - 원자성 : 변경하던가 하지 않던가
  - 일관성 : 한번 변경되면 유지되어야함
  - 고립성 : 다른 세션이 간섭하면 안됨
  - 영속성 : 내가 한번 변경한 데이터는 그대로 유지됨
