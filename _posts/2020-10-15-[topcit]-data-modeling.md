# 데이터 모델링
- 현실세계를 추상화, 단순화, 명확화 하기 위해 일정한 표기법에 의해 표현하는 기술
- 요구사항분석 -> 개념적 모델링 -> 논리적 모델링 -> 물리적 모델링

| 구분 | 내용 |
| :---: | :--- |
| 추상화 | 현실 세계를 일정한 형식에 맞추어 표현 |
| 단순화 | 현실 세계를 약속된 규약이나 제한된 표기법관 언어로 표현 |
| 명확화 | 누구나 이해하기 쉽도록 모호함을 제거하고 정확하게 현상을 기술 |


### 개념적 모델링
- 현실세계에 나타난 것을 구조화시키는 단계(추상적으로 개념화한다.)
- 조직, 사용자의 데이터 요구사항을 분석하고 찾는 것으로 시작
- 어떤 데이터가 중요하고 어떤 형태로 유지되어야 하는지 결정
- 핵심 엔터티와 그들 간의 관계를 발견하는 단계
- 엔터티, 엔터티의 식별자, 엔터티 간 관계, 관계의 대응 수 및 차수, 엔터티의 속성등이 도출됨
- 조직, 사용자의 데이터 요구사항을 분석하고 찾는 것으로 시작

| 같은 말 | 다른 표현 |
| ---: | :--- |
| 엔터티 | 객체 |
| 릴레이션 | 테이블 |
| 속성 or 열 | 컬럼 |
| 전체 데이터 | 인스턴스 |
| 하나의 데이터 행 | 튜플 |
| 차수 | 컬럼 수 |

### 논리적 모델링
- 데이터 모델링에서 가장 핵심이 되는 단계
- 비즈니스 정보의 논리적 구조와 규칙을 명확하게 만드는 단계
- 사람의 이해를 위한 개념적 설계 결과를 데이터베이스 저장이 용이한 논리적 구조로 변환하는 단계
- 데이터를 효율적으로 저장하기위한 정규화가 수행되는 단계
- 주로 관계형 모델리 사용됨
- 개념적 모델링의 산출물 ERD를 테이블 구조도로 변환하는 단계(테이블명, 기본키, 외래키 등이 정의됨)

### 물리적 모델링
- 논리적 데이터 모델링이 데이터 저장소로서 어떻게 컴퓨터 하드웨어 시스템에 표현 될 것인가를 다루는 단계
- 데이블, 컬럼 등으로 표현되는 물리적인 저장 구조, 사용될 저장장치, 자료를 추출하기 위해 사용될 접근 방법이 결정됨
- 가장 중요하게 고려해야하는 요소는 __성능__ 이며 반정규화가 검토되는 단계
- 열의 데이터 형식, 제약조건, 인덱스가 정의됨
