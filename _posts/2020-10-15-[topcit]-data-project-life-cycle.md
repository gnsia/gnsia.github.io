프로젝트 생명주기 내에서 데이터 모델링은 분석단계와 설계단계로 명확하게 정의 될 수 있다. ~~고 하십니다...~~

### 정보공학이나 구조적 모형에 기반한 방법론
- 분석단계에서 업무중심의 논리적인 데이터 모델링을 수행
- 설계단계에서 하드웨어와 성능을 고려한 물리적인 데이터 모델링을 수행

### 나선형 모델
- 업무크기에 따라 논리적 데이터 모델링과 물리적 데이터 모델링이 분석설계단계 양쪽에서 수행
- 비중은 분석단계에서 논리적인 데이터 모델이 더 많이 수행되는 형태

### 프로젝트 생명주기와 데이터베이스 설계

|     단계      | 데이터베이스 설계      | 어플리케이션 설계  |  
| :---: | :---: | :---: |  
| 전환/이행    | DB 전환               | AP 설치   |  
| 테스트       | DB 튜닝              | AP 테스트  |
| 개발        | DB 구축, 변경, 관리     | AP 개발   |
| 설계        | 물리 데이터 모델링       | AP 설계    |
| 분석        | 논리 / 개념 데이터 모델링 | AP 테스트   |

- 데이터 축과 어플리케이션 축으로 구분이 되어 프로젝트가 진행된다.
- 각각에 도출된 사항은 상호검증을 지속적으로 수행하여 단계별 완성도를 높이게 된다.
- 단, __객체 지향 개념__ 은 데이터와 프로세스를 한꺼번에 바라보면서 모델링을 전개하므로 데이터 모델링과 프로세스 모델링을 구분하지 않고 일체형으로 진행한다.
