# 데이터 분석 개요

## 데이터 처리 과정

- 일반적으로 기업에선 데이터 분석을 위하여 DW나 DM을 통해 분석데이터를 구성합니다.
- 신규데이터나 DW에 없는 데이터는 기존 운영시스템(Legacy)에서 가져오기보다는 운영 시스템에서 임시로 데이터를 저장하는 스테이징영역(Staging area)에서 데이터를 전처리하여 운영데이터 저장소(ODS)에 저장된 데이터를 DW와 DM와 결합해서 데이터를 구성합니다.

## 데이터 시각화

- 시각화는 가장 낮은 수준의 분석이지만 잘 사용하면 복잡한 분석보다도 더 효율적입니다.
- 대용량 데이터를 다루는 빅데이터 분석에서 시각화는 필수입니다.
- 탐색적 분석을 할 때 시각화는 필수입니다.

## 탐색적자료분석(EDA)

- 탐색적 분석 : 다양한 차원과 값을 조합해 가며 특이한 점이나 의미 있는 사실을 도출하고 분석의 최종 목적을 달성해가는 과정입니다.

- 탐색적 분석의 효율성 확대 방법
  - 의미가 있을 것으로 판단되는 변수집단과 아닌 집단을 1차적으로 구분합니다.
  - 전체 변수가 300개 일 경우, 의미가 있는 1차 집단 50개, 2차 집단이 100개, 의미가 없는 집단이 150개 정도로 구분됩니다.

## 통계 분석

- 통계 : 어떤 현상을 종합적으로 한 눈에 알아보기 쉽게 일정한 체계에 따라 숫자와 표, 그림의 형태로 나타내는 것이라 부릅니다.

- 기술통계 : 모집단으로부터 표본을 추출하고 표본이 가지고 있는 정보를 쉽게 파악할 수 있도록 데이터를 정리하거나 요약하기위해 하나의 숫자로 또는 그래프의 형태로 표현하는 절차입니다.

- 추측통계 : 모집단으로부터 추출된 표본의 표본통계량으로부터 모집단의 특성인 모수에 관해 통계적으로 추론하는 절차입니다.

- 활용분야
  - 정부의 경제정책수립과 평가의 근거자료로 활용(통계청의 실업률, 고용률, 물가지수)
  - 농업(가뭄, 수해 또는 병충해 등에 강한 품종의 개발 및 개량)
  - 의학(의학적 치료방법의 효과나 신약 개발을 위한 임상실험의 결과 분석)
  - 경영(제품 개발, 품질관리, 시장조사, 영업관리 등에 활용)
  - 스포츠(선수들의 체질향상 및 개선, 경기 분석과 전략분석, 선수평가와 기용 등)
